# 📚 自习室预约系统 - 本地测试指南

这是一份简单易懂的本地开发和测试指南，帮助你快速找到各个端在本地运行的访问地址。

---

## 🖥️ 1. 前端访问地址 (Vue 3)

目前我们的系统是**用户端和管理员端融合在一起的**（即通过登录不同的账号来展示不同的界面和权限）。

*   **✅ 官方主测试地址**: [http://localhost:5173/](http://localhost:5173/)
*   **⚠️ 为什么 `5174` 端口老是掉线？**
    *   Vite 前端框架默认固定使用 `5173` 端口。当你看到 `http://localhost:5174/` 时，说明你不小心在后台**运行了两次**启动命令（第二个进程被迫使用了 5174 端口）。
    *   作为多余出来的“影子进程”，`5174` 极易因为终端冲突、资源回收或者主进程覆盖而掉线。
    *   **解决办法**：以后本地测试**只认准 `5173`** 即可！把多余的终端窗口关掉，保证只有一个前端在运行，它就会稳如泰山。

## ⚙️ 2. 后端 API 接口地址 (FastAPI)

后端启动后，你可以通过以下地址查看、调试系统所有的后台数据。

*   **📖 交互式 API 接口文档 (极力推荐)**: [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)
    *   *说明：这是 FastAPI 框架最牛的功能，它会自动生成所有接口的可视化文档。你可以直接在这里点击 "Try it out" 模拟发送请求，测试后端逻辑。*
*   **🔗 基础请求地址**: `http://127.0.0.1:8000`
    *   *说明：本地开发时，前端 Axios 就是向这个基础地址发送数据的。*

## 🚀 3. 如何在本地完整启动项目？

如果你刚打开电脑，想要运行项目，请打开**两个独立**的终端窗口，分别执行：

### 🟢 终端 1：启动后端
```bash
cd server
..\venv\Scripts\python.exe -m uvicorn main:app --reload --port 8000
```
*(看到 `Application startup complete` 代表后端启动成功)*

### 🔵 终端 2：启动前端
```bash
cd client
npm run dev
```
*(看到 `Local: http://localhost:5173/` 代表前端启动成功)*
